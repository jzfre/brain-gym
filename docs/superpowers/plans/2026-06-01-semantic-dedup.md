# Semantic Duplicate Prevention (pgvector) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Upgrade problem duplicate-prevention from exact SHA-256 hashing to semantic similarity using OpenAI embeddings stored in pgvector, with a reactive gate plus proactive steering.

**Architecture:** Embeddings live in a `vector(1536)` column on `Problem` inside the existing Postgres (pgvector image). On generation, the model produces a candidate, we embed it and KNN-search prior problems of the same type; if cosine similarity ≥ threshold we regenerate (feeding the colliding neighbors back as avoidance), up to a retry budget, then save the least-similar candidate flagged as a near-duplicate. Raw vector SQL is confined to one helper and one write line; all decision logic is a pure, dependency-injected function.

**Tech Stack:** Next.js 15 App Router, Prisma 5 (+ `Unsupported("vector")`), pgvector, OpenAI `text-embedding-3-small`, Vitest, Zod.

**Spec:** `docs/superpowers/specs/2026-06-01-semantic-dedup-design.md`

**Branch:** `feat/semantic-dedup` (already created; the design doc is its first commit).

```typescript
type EmbeddingTextInput = {
  title: string;
  userVisiblePrompt: string;
  tags: string[];
  questions?: Array<{ stimulus: string; questionStem: string }>;
};

// The single source of truth for what text represents a problem. Used by the
// dedup gate. Keep stable: changing it invalidates comparisons against older
// embeddings.
export function problemEmbeddingText(p: EmbeddingTextInput): string {
  const tagStr = [...p.tags].map((t) => t.trim().toLowerCase()).sort().join(", ");
  const lsat =
    p.questions && p.questions.length > 0
      ? "\n" + p.questions.map((q) => `${q.stimulus}\n${q.questionStem}`).join("\n")
      : "";
  return [p.title.trim(), p.userVisiblePrompt.trim(), tagStr].join("\n") + lsat;
}
```

```typescript
import { getOpenAI } from "@/lib/openai/client";
import { getConfig } from "@/lib/config";

export type Embedder = (text: string) => Promise<number[]>;

// Returns the embedding vector, or null on any failure. Embedding must never
// block a problem from being saved, so callers treat null as "skip the
// semantic gate for this one".
export async function embedText(text: string, embed?: Embedder): Promise<number[] | null> {
  try {
    if (embed) return await embed(text);
    const cfg = getConfig();
    const resp = await getOpenAI().embeddings.create({ model: cfg.embedding.model, input: text });
    return resp.data[0].embedding;
  } catch {
    return null;
  }
}
```

```typescript
import { Prisma } from "@prisma/client";
import { prisma } from "@/lib/db/client";

export type SimilarProblem = {
  id: string;
  title: string;
  tags: string[];
  similarity: number;
};

// The ONLY place raw vector SQL lives. Returns the k nearest prior problems of
// the given exercise type by cosine similarity (1 = identical), closest first.
// Degrades to [] on any query error so the caller falls back to hash-only.
export async function findSimilarProblems(args: {
  exerciseTypeId: string;
  embedding: number[];
  k: number;
}): Promise<SimilarProblem[]> {
  const literal = `[${args.embedding.join(",")}]`;
  try {
    const rows = await prisma.$queryRaw<SimilarProblem[]>(Prisma.sql`
      SELECT "id", "title", "tags",
             1 - ("embedding" <=> ${literal}::vector) AS "similarity"
      FROM "Problem"
      WHERE "exerciseTypeId" = ${args.exerciseTypeId}
        AND "embedding" IS NOT NULL
      ORDER BY "embedding" <=> ${literal}::vector
      LIMIT ${args.k}
    `);
    return rows.map((r) => ({ ...r, similarity: Number(r.similarity) }));
  } catch {
    return [];
  }
}
```

Create `lib/memory/dedup.ts`:

```typescript
import type { GenerateInput, GenerateResult } from "@/lib/exercises/types";
import type { SimilarProblem } from "./similarity";

export type DedupDeps = {
  generate: (input: GenerateInput) => Promise<GenerateResult>;
  embed: (text: string) => Promise<number[] | null>;
  findSimilar: (embedding: number[]) => Promise<SimilarProblem[]>;
  embeddingText: (problem: GenerateResult["problem"]) => string;
};

export type DedupConfig = { threshold: number; maxRetries: number };

export type DedupOutcome = {
  result: GenerateResult;
  embedding: number[] | null;
  nearestSimilarity: number | null;
  isNearDuplicate: boolean;
};

// Reactive gate + proactive steering in one loop. Generates a candidate, embeds
// it, and checks cosine similarity against prior problems. Under threshold →
// accept. Over threshold → remember the best (least-similar) candidate and
// regenerate with the colliding neighbors appended to the avoidance hint. After
// the retry budget, save the best candidate flagged as a near-duplicate. If
// embedding fails, accept immediately (hash layer still applies downstream).
export async function generateUniqueProblem(args: {
  input: GenerateInput;
  deps: DedupDeps;
  config: DedupConfig;
}): Promise<DedupOutcome> {
  const { input, deps, config } = args;
  let avoidance = input.avoidanceHint;
  let best: DedupOutcome | null = null;

  for (let attempt = 1; attempt <= config.maxRetries; attempt++) {
    const result = await deps.generate({ difficulty: input.difficulty, avoidanceHint: avoidance });
    const embedding = await deps.embed(deps.embeddingText(result.problem));

    if (!embedding) {
      return { result, embedding: null, nearestSimilarity: null, isNearDuplicate: false };
    }

    const neighbors = await deps.findSimilar(embedding);
    const maxSim = neighbors.length > 0 ? neighbors[0].similarity : 0;

    if (maxSim < config.threshold) {
      return { result, embedding, nearestSimilarity: maxSim, isNearDuplicate: false };
    }

    if (best === null || maxSim < (best.nearestSimilarity ?? 1)) {
      best = { result, embedding, nearestSimilarity: maxSim, isNearDuplicate: true };
    }

    avoidance = {
      ...avoidance,
      recentTitles: [...avoidance.recentTitles, ...neighbors.map((n) => n.title)],
      recentTags: [...avoidance.recentTags, ...neighbors.flatMap((n) => n.tags)]
    };
  }

  // Every attempt exceeded the threshold; `best` is guaranteed set here.
  return best as DedupOutcome;
}
```

```markdown
## Duplicate prevention

Two layers stop repeated problems:

1. **Exact hash** — a SHA-256 over the normalized title, prompt start, and tags
   (`lib/memory/uniqueness.ts`), enforced by a unique constraint per exercise type.
2. **Semantic gate** — each generated problem is embedded (OpenAI
   `text-embedding-3-small`) and compared by cosine similarity against prior
   problems of the same type via `pgvector` (`lib/memory/similarity.ts`). If the
   nearest prior problem is at or above `DEDUP_SIMILARITY_THRESHOLD`, generation
   retries (up to `DEDUP_MAX_RETRIES`), feeding the colliding problems back as
   things to avoid. If every retry is still too similar, the least-similar
   candidate is saved and flagged `near-dup` (visible in History and
   `/admin/problems`).

Embedding failures never block generation: a problem still saves, just without an
embedding (and so without the semantic check for that one).
```

## Self-review notes (for the implementer)

- **DRY:** `problemEmbeddingText` is the single definition of embed text; `findSimilarProblems` is the single place vector SQL lives. Don't inline either.
- **YAGNI:** No backfill script (data is wiped); no attempt/evaluation embeddings (next project).
- **Embedding writes:** always via `tx.$executeRaw` with a `[...]::vector` literal — the typed Prisma client cannot write the `Unsupported("vector")` column.
- **Non-fatal embeddings:** `embedText` returns `null` on failure and the loop accepts immediately; never let an embedding error 502 a generation.
